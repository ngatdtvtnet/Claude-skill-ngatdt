📁 Cấu trúc thư mục Skill

Một Skill về bản chất chỉ là một thư mục chứa các file:

ten-skill-cua-ban/
├── SKILL.md          ← Bắt buộc (file chính)
├── scripts/          ← Tùy chọn (mã thực thi)
├── references/       ← Tùy chọn (tài liệu tham khảo)
└── assets/           ← Tùy chọn (mẫu, font, icon)
⚠️
Quy tắc đặt tên:
• Thư mục: dùng kebab-case → viettel-report-gen ✅
• File chính: phải đúng SKILL.md (phân biệt hoa thường)

📄 File SKILL.md — Trái tim của Skill

SKILL.md gồm 2 phần chính:

1. YAML Frontmatter (Phần khai báo)

---
name: viettel-report-gen
description: Tạo báo cáo theo chuẩn Viettel Networks.
  Sử dụng khi người dùng yêu cầu "tạo báo cáo",
  "viết tờ trình", hoặc "tổng hợp số liệu".
---

2. Nội dung Markdown (Hướng dẫn chi tiết)

Phần bên dưới frontmatter chứa toàn bộ hướng dẫn: quy trình, ví dụ, lưu ý, xử lý lỗi.

🎯 3 Nguyên tắc Thiết kế Cốt lõi

Nguyên tắcGiải thíchLợi ích
Progressive Disclosure Tải thông tin theo 3 cấp: YAML → SKILL.md → files đính kèm Tiết kiệm token, tải đúng lúc cần
Composability Skill hoạt động tốt cùng các Skill khác Không xung đột, kết hợp linh hoạt
Portability Chạy giống nhau trên Claude.ai, Claude Code, API Tạo 1 lần, dùng mọi nơi

🧠 Hệ thống 3 Cấp — Progressive Disclosure

  1. Cấp 1 — YAML frontmatter: Luôn được tải vào system prompt. Claude dùng phần này để quyết định "có nên dùng Skill này không?"
  2. Cấp 2 — Nội dung SKILL.md: Chỉ tải khi Claude xác định Skill phù hợp. Chứa đầy đủ hướng dẫn, quy trình, ví dụ.
  3. Cấp 3 — Tệp đính kèm: Scripts, references, assets — Claude chỉ đọc khi thực sự cần thiết.
💡
Cơ chế này giúp giảm 50-70% token so với việc nạp mọi thứ cùng lúc.

🍳 So sánh: MCP vs Skill

Hãy tưởng tượng bạn đang nấu ăn:

🏪 MCP = Nhà bếp

Cung cấp công cụ, nguyên liệu, thiết bị.

→ Claude có thể làm gì

📋 Skill = Công thức nấu

Hướng dẫn từng bước để tạo món ăn.

→ Claude nên làm như thế nào

MCP (Kết nối)Skill (Kiến thức)
Kết nối Claude với dịch vụ (Notion, Asana...)Dạy Claude cách sử dụng dịch vụ hiệu quả
Cung cấp truy cập dữ liệu thời gian thựcNắm bắt quy trình & phương pháp tốt nhất

📋 Tóm tắt Chương 1

  1. Skill = Thư mục chứa SKILL.md (bắt buộc) + scripts, references, assets (tùy chọn)
  2. SKILL.md = YAML frontmatter (khai báo) + Markdown (hướng dẫn)
  3. 3 nguyên tắc: Progressive Disclosure, Composability, Portability
  4. MCP + Skill = Bộ đôi hoàn hảo: công cụ + quy trình
